28// to attempt to re-establish a connection to the game server.
29// We'll use 5 seconds here as it's a more than reasonable enough time for it to receive a packet, even with lag, while not being
30// too excessive.
31// If the client disconnects in this time, this timer will be reset, so there's no need to account for this.
32// Officially it Sleep()s for 1 second prior to even sending WIZ_VERSION_CHECK packet we need to wait for, so there's not really
33// a comparable official limit. All this accomplishes is reducing the number of connection attempts, not preventing overlaps.
