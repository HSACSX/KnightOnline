Detailed Description

Base class representing an active audio playback instance.

AudioHandle encapsulates playback state, timing information, and configuration shared between buffered and streamed audio playback. Concrete subclasses manage backend-specific resources.

◆ IsManaged

std::atomic<bool> AudioHandle::IsManaged

Indicates whether this handle is currently managed by the audio thread.

The flag is set when the handle is queued for decoding/playback in the AudioThread (i.e. via AudioThread::Add) and cleared when the handle is removed from the AudioThread.

CN3SndObj::Tick() checks this flag to detect when the handle is no longer active; if false, the object releases its reference to the handle, restoring it to its associated pool for reuse.

◆ Settings

std::shared_ptr<SoundSettings> AudioHandle::Settings

Shared sound settings for this audio handle.

Contains volume settings. While the handle can access and modify these settings, they are not exclusively owned by the handle. For example, if the volume is changed, the new value persists in CN3SndObj for subsequent playback of the same sound, ensuring consistency across uses. This allows both the audio engine and the handle instance to reference the same configuration.

e_SndState AudioHandle::State

Current sound playback state.

Indicates the lifecycle state of this audio handle, as defined by e_SndState. Typical states include initial, playing, paused, or stopped. This is used by the audio engine to manage playback progression, transitions, and cleanup.
